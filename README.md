Платформа для поиска и организации событий, где пользователи могут создавать мероприятия, оставлять заявки на участие и формировать подборки. Проект реализован в **микросервисной архитектуре** с разделением на основной сервис управления событиями и сервис сбора статистики просмотров.

## **🛠️ Стек технологий**

Java 21, Spring Boot 3.3.2, Spring Data JPA, PostgreSQL 16, Docker, Maven, Lombok 1.18.32, JUnit 5, Mockito 5.11.0, Testcontainers 1.15.3, Jakarta Validation 3.0.0, Checkstyle 3.3.1, SpotBugs 4.8.5.0, JaCoCo

## **🚀 Особенности проекта**

**📅 Управление событиями**

- Создание, редактирование, публикация и поиск мероприятий
- Фильтрация по дате, категории, доступности мест

**📝 Система заявок**

- Подача, подтверждение и отклонение заявок на участие
- Учёт лимитов мест при подтверждении

**⭐ Рейтинги**

- Возможность ставить лайки и дизлайки событиям

**📂 Подборки**

- Формирование тематических подборок событий
- Возможность закрепления подборок

**📊 Статистика**

- Сбор и агрегация данных о просмотрах событий
- Учёт уникальных посетителей

## **🏗️ Архитектура и технические решения**

- **Микросервисная архитектура:**
  - ewm-service (основной API)
  - stats-server (сервис статистики)
  - Отдельные базы PostgreSQL для каждого сервиса
- Взаимодействие сервисов через REST API
- Многослойная архитектура внутри каждого сервиса: Controller → Service → Repository
- Использование DTO для изоляции API от доменной модели
- Глобальная обработка исключений через @ControllerAdvice
- Управление транзакциями с @Transactional
- Покрытие кода тестами (JUnit 5, Mockito, Testcontainers)
- Статический анализ кода: Checkstyle, SpotBugs
- Maven-профили: check и coverage

## **💡 Мой вклад**

Проект разработан полностью самостоятельно в рамках учебного трека. Ключевые реализованные задачи:

**Архитектура и модульность**

- Спроектировал микросервисную архитектуру с разделением на основной сервис и сервис статистики
- Настроил взаимодействие между сервисами через REST
- Организовал модульную структуру Maven (родительский POM + дочерние модули)

**Бизнес-логика событий и заявок**

- Реализовал полный цикл управления событиями (создание, редактирование, публикация, модерация)
- Добавил систему заявок с проверкой лимитов и статусами (PENDING, CONFIRMED, REJECTED, CANCELED)
- Написал сложные запросы для фильтрации событий по множеству параметров

**Подборки и рейтинги**

- Создал функциональность формирования тематических подборок
- Реализовал систему лайков и дизлайков к событиям

**Статистика**

- Разработал отдельный микросервис для сбора статистики просмотров
- Реализовал эндпоинты для сохранения и агрегации данных
- Добавил подсчёт уникальных посетителей

**Тестирование и качество кода**

- Покрыл основные сценарии юнит- и интеграционными тестами
- Использовал Testcontainers для тестирования с реальной БД
- Настроил Checkstyle и SpotBugs для поддержания чистоты кода

**Контейнеризация**

- Написал Dockerfile для каждого микросервиса
- Создал docker-compose для оркестрации всех сервисов и баз данных

## **📌 Выводы**

В ходе разработки проекта освоил проектирование микросервисной архитектуры с разделением на основной сервис управления событиями и сервис статистики. На практике отработал взаимодействие между независимыми сервисами через REST API и организацию отдельных баз данных для каждого компонента.

Применение Testcontainers позволило обеспечить качественное интеграционное тестирование с реальными базами данных. Контейнеризация через Docker обеспечила удобство развертывания и воспроизводимость окружения.

В результате создано готовое к деплою микросервисное приложение с гибкой системой фильтрации событий, модерацией и сбором статистики.

  

## **▶️ Системные требования и запуск**

### **Структура сервисов**

| **Сервис** | **Назначение** | **Порт** |
|----|----|----|
| ewm-service | Основной API приложения | 8080 |
| stats-server | Сервис сбора статистики | 9090 |
| ewm-db | PostgreSQL для основного сервиса | 5432 |
| stats-db | PostgreSQL для сервиса статистики | 5433 |

### **Требования**

- **Java 21** (проект использует возможности, доступные только в этой версии)
- **Maven 3.9+**
- **Docker 20.10+** (для запуска всех сервисов через docker-compose)
- **Порты 8080, 9090, 5432, 5433** — должны быть свободны

### **Запуск**

bash

```javascript
# Клонирование репозитория
git clone https://github.com/your-username/explore-with-me.git
cd explore-with-me

# Сборка проекта
mvn clean install

# Запуск всех сервисов через Docker Compose
docker-compose up
```

Приложения будут доступны:

- Основной API: http://localhost:8080
- Сервис статистики: http://localhost:9090
